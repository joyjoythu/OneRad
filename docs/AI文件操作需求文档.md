# AI 辅助文件操作需求文档

## 1. 背景与动机

OneRad 当前已支持通过 DeepSeek API 进行医学影像分析。用户希望进一步扩展能力：通过自然语言描述需求，由 AI 自动规划并执行文件整理、移动、复制等操作，降低手动处理医学影像文件和临床数据的门槛。

## 2. 目标

为 OneRad 增加一个**AI 辅助文件操作**模块，实现：

- 用户用自然语言描述文件处理需求
- AI 将需求解析为结构化、可解释的操作计划
- 用户在执行前审阅并确认计划
- 系统仅执行预定义的安全文件操作
- 操作结果可追踪、可回滚

## 3. 用户故事

### 3.1 文件整理

> 作为研究人员，我希望说“把 MRI 文件夹里所有 nii 文件按患者 ID 分到不同子文件夹”，AI 就能帮我整理，这样我不用手动拖拽几百个文件。

### 3.2 数据归档

> 作为研究人员，我希望说“把已完成分析的患者数据备份到 output/归档/ 目录”，AI 生成操作计划并让我确认后再执行。

### 3.3 批量重命名

> 作为研究人员，我希望说“把所有文件名里的空格改成下划线”，AI 给出重命名清单，我确认后再应用。

## 4. 功能需求

### 4.1 需求输入

- 提供一个文本输入框，用户输入自然语言需求
- 支持引用当前项目目录作为默认上下文
- 可选：提供文件列表快照，帮助 AI 理解当前目录结构

### 4.2 操作计划生成

- 调用 DeepSeek API 解析用户需求
- AI 返回 JSON 格式的操作计划数组
- 每个操作包含：
  - `action`: 操作类型（move | copy | rename | mkdir）
  - `source`: 源路径
  - `target`: 目标路径
  - `reason`: AI 执行该操作的原因说明

### 4.3 计划审阅

- UI 以表格或卡片形式展示操作计划
- 每个操作显示：类型、源文件、目标位置、原因
- 提供“确认执行”和“取消”按钮
- 用户可以在执行前编辑或删除某个操作

### 4.4 安全执行

- 仅允许白名单内的操作类型：move、copy、rename、mkdir
- **禁止 delete 操作**
- 所有路径必须解析在项目目录范围内，禁止越界（如 `../../../etc`）
- 目标路径已存在时，必须提示用户选择覆盖或跳过
- 所有操作在执行前记录到日志

### 4.5 结果反馈

- 执行完成后显示成功/失败列表
- 失败时显示具体原因和错误路径
- 提供“撤销上次操作”入口（通过备份实现）

### 4.6 日志与审计

- 每次文件操作记录到项目日志（时间、操作、源、目标、执行结果）
- 日志文件保存在项目目录下，便于回溯

## 5. 非功能需求

### 5.1 安全性

- AI 不直接执行代码，只生成结构化计划
- 所有实际文件操作由预定义函数执行
- 路径必须做规范化处理并校验沙箱范围
- 敏感路径（系统目录、用户根目录等）必须拒绝

### 5.2 可解释性

- 每个操作必须有 `reason` 说明
- 用户必须确认后才能执行
- 计划展示要清晰易懂，不能隐藏风险操作

### 5.3 可扩展性

- 操作类型设计为可扩展，后续可加入 compress、verify 等
- 提示词模板独立管理，便于调整

### 5.4 兼容性

- 与现有项目系统（`ProjectStore`、`LLMClient`）集成
- 支持 Windows 和 Linux/macOS 路径处理

## 6. 工作流程

```
用户输入需求
    ↓
系统调用 DeepSeek API 生成操作计划（JSON）
    ↓
系统校验计划合法性（路径沙箱、操作白名单）
    ↓
UI 展示计划，等待用户确认
    ↓
用户确认 / 编辑 / 取消
    ↓
系统执行操作，记录日志
    ↓
展示执行结果
```

## 7. 边界与约束

### 7.1 在范围内

- 项目目录内的文件移动、复制、重命名
- 新建目录
- 基于自然语言的批量操作

### 7.2 不在范围内（后续可考虑）

- 删除文件操作
- 修改文件内容
- 访问项目目录之外的文件
- 自动执行无需确认
- 网络文件操作（FTP、云盘等）

## 8. 界面原型描述

在 OneRad 主界面新增一个标签页或独立区块：

- **需求输入区**：多行文本框 + “生成计划”按钮
- **计划展示区**：表格展示 action / source / target / reason
  - 每行可删除
  - 目标路径可编辑
- **操作按钮**：确认执行 / 取消
- **结果区**：执行成功/失败列表

## 9. 验收标准

1. 用户输入“把 test 目录下的所有 .txt 文件复制到 backup 目录”，系统能生成正确的 copy 计划
2. 计划执行前必须展示并等待用户确认
3. 涉及项目目录外的路径时，系统拒绝执行并给出提示
4. 不允许执行 delete 操作
5. 所有操作记录到项目日志
6. 执行失败后，已成功的操作不会被回滚，但失败原因清晰展示
7. 与现有 DeepSeek API 配置共用，不需要额外配置

## 10. 依赖与前置条件

- 复用现有 `LLMClient` 调用 DeepSeek API
- 复用现有项目目录结构
- 新增 `app/actions.py` 模块
- 在 `app/ui.py` 中新增操作界面

## 11. 风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| AI 生成错误路径 | 高 | 路径沙箱校验 + 用户确认 |
| AI 建议删除文件 | 高 | 操作白名单禁止 delete |
| 用户误操作覆盖文件 | 中 | 目标存在时提示 + 操作前备份 |
| 大目录下计划过多 | 低 | 分页展示 + 限制单次最大操作数 |
