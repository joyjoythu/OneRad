# OneRad UI 视觉与布局改造设计文档

## 背景与目标

当前 `app/ui.py` 使用 Gradio 默认样式搭建的 OneRad 界面在视觉层级、卡片化布局、图标精致度上与目标参考图差距较大。本次改造目标是在**不改动业务功能**的前提下，复刻参考图的现代医疗科技风格，让配置界面更清晰、专业、易用。

## 范围

- 仅修改 `app/ui.py` 的视觉结构与样式，以及可能新增的辅助 CSS/图标资源。
- 保持现有事件绑定、项目存储、分析流程逻辑不变。
- 不涉及新增后端接口、不涉及更换 UI 框架。

## 方案选择

选用 **方案 A：Gradio + 自定义 CSS**。理由：
- 改动最小，保持现有 Gradio Blocks 架构。
- 通过全局 CSS、元素类名和少量 `gr.HTML` 即可实现参考图的卡片化布局和品牌头部。
- 维护成本低，未来功能扩展仍可沿用 Gradio 组件。

未选用方案 B/C 的原因：
- B 对复杂区块控制更强，但头部/侧边栏用 HTML 注入后与 Gradio 表单的对齐、状态同步更繁琐。
- C 需要新增前后端接口层，超出“只改视觉和布局”的范围。

## 设计细节

### 1. 整体布局

> 以下 ASCII 图仅为结构示意；实际界面使用内联 SVG 图标，不显示 emoji。

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo] OneRad          医学影像智能分析平台       v2.0 Ready │  ← 品牌头部
├──────────────┬──────────────────────────────────────────────┤
│              │                                              │
│  项目管理    │  当前项目：肺结节分析                          │
│  + 新建项目  │  配置影像路径、临床表格与模型参数              │
│  选择项目 ▾  │                                              │
│  [OK] 已就绪 │  ── 数据源 ─────────────────────────────     │
│              │  影像文件夹路径    临床表格路径              │
│              │                                              │
│              │  ── 分析配置 ────────────────────────────    │
│              │  输出目录 / 模态 / 协变量                      │
│              │                                              │
│              │  ── AI 模型配置 ─────────────────────────    │
│              │  DeepSeek API Key    模型                     │
│              │                                              │
│              │              [保存配置]  [开始分析]           │
│              │                                              │
│              │  ── 运行日志 ────────────────────────────    │
│              │  [OK] [19:42:01] 项目加载完成                │
│              │  [OK] [19:42:03] 检测到 128 个病例           │
│              │  [>>] [19:42:05] 等待开始分析...              │
│              │                                              │
└──────────────┴──────────────────────────────────────────────┘
```

- 顶部固定品牌栏：左侧 Logo 图标 + OneRad 标题 + 副标题，右侧 `v2.0` 与 `Ready` 徽章。
- 下方两栏：左侧项目卡片固定宽度（约 320px），右侧配置主卡片自适应剩余宽度。
- 取消原 hamburger 侧边栏切换，侧边栏常驻显示（与参考图一致）。
- 页面背景使用浅灰 `#f5f6f8`，所有内容置于白色卡片内。

### 2. 视觉风格

| 元素 | 取值 |
|------|------|
| 主色/强调色 | `#2563eb`（深蓝）用于图标、徽章、主按钮 |
| 成功/就绪 | `#16a34a` / `#dcfce7` |
| 背景色 | `#f5f6f8` |
| 卡片背景 | `#ffffff` |
| 卡片阴影 | `0 1px 3px rgba(0,0,0,0.06)` |
| 卡片圆角 | `12px` |
| 标题文字 | `#1f2937`，字号 18–20px，font-weight 700 |
| 次级文字 | `#6b7280`，字号 12–13px |
| 输入框边框 | `#e5e7eb`，圆角 `8px`，背景 `#f9fafb` |
| 分隔线 | 1px `#e5e7eb` |

### 3. 品牌头部

使用 `gr.HTML` 渲染固定头部 HTML：
- 左侧：圆角方形 Logo（心电波形/医疗十字 SVG），紧邻 OneRad 标题和“医学影像智能分析平台”副标题。
- 右侧：`v2.0` 灰色徽章 + `Ready` 绿色徽章。

### 4. 左侧项目卡片

- 标题带文件夹 SVG 图标。
- “+ 新建项目”按钮：浅灰背景、圆角、居中。
- “选择项目”使用 `gr.Dropdown` 并加样式使其看起来像参考图的下拉框。
- “项目状态”区域：根据配置完整性显示不同状态：
  - 已就绪：绿色勾选图标 + “配置已保存，可开始分析”。
  - 等待配置：蓝色信息图标 + “请填写影像路径和临床表格”。
  - 配置错误：红色错误图标 + 错误说明。

### 5. 右侧配置主卡片

- 卡片头部：大标题 “当前项目：{name}”、副标题。**“Auto 模式”蓝色徽章当前未实现，标记为后续增强。**
- 数据源区块：标题带数据源图标；影像文件夹路径、临床表格路径两个输入框并排。**输入框内左侧图标当前未实现，标记为后续增强。**
- 分析配置区块：输出目录、模态下拉、协变量输入。
- AI 模型配置区块：DeepSeek API Key 密码框、模型选择/文本框。**显示/隐藏眼睛图标切换当前未实现，标记为后续增强。**
- 操作栏：右侧对齐，“保存配置”为次按钮，“开始分析”为主按钮。
- 运行日志区块：标题带日志图标 + “实时输出”蓝色小标签；日志使用等宽字体列表。**底部下载日志按钮当前未实现，标记为后续增强。**

### 6. 图标方案

使用内联 SVG 图标，避免 emoji 在不同平台的渲染差异。图标尺寸统一 16–20px，颜色继承主色或次级灰。

已实现图标：
- OneRad Logo（心电波形 + 圆角方块）
- 文件夹（项目管理、数据源）
- 表格/文档（临床表格路径）
- 齿轮（分析配置）
- 地球/AI 芯片（AI 模型配置）
- 文档/终端（运行日志）
- 勾选圆、信息圆、错误圆（状态）

已定义但尚未接入的图标：
- 眼睛/眼睛关闭（密码显示切换）

### 7. Gradio 实现要点

- 使用 `gr.Blocks(title="OneRad")` 创建主界面，**不在 `gr.Blocks()` 构造函数中通过 `css` 参数注入样式**。
- 在启动时通过 `demo.launch(css=CUSTOM_CSS)` 的 `css` 参数传入全局自定义 CSS。
- 通过 `elem_classes` 为卡片、输入框、按钮等组件添加自定义类名。
- 品牌头部和区块标题使用 `gr.HTML` 渲染带图标的 HTML。
- 保持现有的事件绑定：项目选择、新建/删除项目、保存配置、运行分析。
- 日志区域仍使用 `gr.Textbox` 或 `gr.HTML`，通过现有 `log` 状态更新。

## 数据流与状态

与现有实现保持一致：
1. `ProjectStore` 管理项目列表、配置加载/保存。
2. `current_project_id` Gradio State 记录当前选中项目。
3. 表单字段与项目配置双向同步。
4. 点击“开始分析”时先保存配置，再调用 `_run_analysis` 生成日志和报告。

## 错误处理

- 保持现有错误提示：未选择项目、路径为空、分析失败等。
- 状态区域根据错误信息显示红色错误状态。
- 日志区域继续展示异常 traceback 或失败信息。

## 测试

- 启动 UI：`python main.py --ui`

- 验证视觉样式在不同窗口尺寸下无明显错位（Gradio 响应式由框架处理）。
- 运行现有测试：`pytest tests/test_ui.py tests/test_projects.py`

## 已实现文件

- `app/ui.py`：主要改造文件。
- `app/ui_style.py`：定义 `CUSTOM_CSS`、SVG 图标与 HTML 辅助函数。

## 决策记录

| 决策 | 选项 | 理由 |
|------|------|------|
| 框架 | 保留 Gradio | 用户要求“只改视觉和布局” |
| 图标 | 内联 SVG | 与参考图一致，跨平台渲染稳定 |
| 侧边栏 | 常驻显示 | 与参考图一致，取消 hamburger |
| 头部 | `gr.HTML` 自定义 | 参考图头部包含 Logo、副标题、多徽章，超出 Gradio 原生能力 |
| CSS 注入 | `demo.launch(css=CUSTOM_CSS)` | 与当前 Gradio 启动方式一致，运行时 `/config` 端点可返回自定义 CSS |
