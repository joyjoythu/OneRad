# 左侧项目侧边栏常驻列表设计

## 背景

OneRad（AutoRadiomics Agent）的 UI 使用 Gradio 构建。当前左侧"项目管理"区域使用 `gr.Dropdown` 选择项目，用户需要先点击下拉框才能看到已有项目，不符合"一眼看到所有项目"的使用习惯。

本设计将左侧项目管理区改为常驻项目列表：所有已创建项目直接显示在侧边栏，点击即可切换，行尾提供删除按钮。

## 目标

- 左侧边栏直接展示所有现有项目，无需点击展开。
- 点击项目行即可切换到对应项目并加载配置。
- 每行最右侧显示删除图标（×），点击后弹出确认框，确认后删除项目。
- 新建项目仍通过"+ 新建项目"按钮展开表单，创建后新项目出现在列表顶部。
- 当前选中的项目行使用蓝色背景高亮。
- 项目数量不受预定义组件数量限制。

## 需求确认

| 问题 | 用户选择 |
|------|----------|
| 展示形式 | 常驻项目列表，不用 Dropdown |
| 每行信息 | 仅显示项目名称 |
| 删除交互 | 行尾 × 按钮 → 确认弹窗 → 删除 |
| 新建表单 | 侧边栏内展开 |
| 选中高亮 | 蓝色背景高亮 |
| 项目数量 | 无明确上限，需支持滚动 |

## 方案对比

### 方案 A：预定义固定行数

在 `gr.Blocks` 中预定义 N 个项目行（如 20 行），每行包含项目按钮和删除按钮。通过 `gr.update` 控制显示/隐藏和文本内容。

- **优点**：纯 Python，无需手写 JS，事件绑定直接。
- **缺点**：项目数量有上限；代码冗余；滚动、空状态、动画不易实现。

### 方案 B：HTML 自定义列表 + JS 通信（已选）

使用 `gr.HTML` 渲染整个项目列表，HTML 内嵌 JS 处理点击事件。通过隐藏的 `gr.Textbox` 作为"事件桥"，把"选择项目 ID"和"删除项目 ID"传回 Python 处理。

- **优点**：项目数量无上限；视觉完全可控，最接近参考图；代码紧凑。
- **缺点**：需要少量 JS；需要隐藏组件做事件桥接。

### 方案 C：原生 Radio + 独立删除

使用 `gr.Radio` 显示项目列表，删除操作通过先选中项目再点击独立删除按钮完成。

- **优点**：实现最简单，无需 JS。
- **缺点**：不能满足行内 × 删除的需求；视觉样式受限。

## 视觉设计

### 整体布局

- 左侧：`min-width: 320px` 的固定宽度侧边栏。
- 右侧：占满剩余宽度的主工作区。
- 侧边栏与主工作区均使用现有的 `onerad-card` 卡片样式。

### 侧边栏结构（从上到下）

1. **标题栏**："项目" + 右上角刷新/新建按钮（保留"+ 新建项目"按钮）。
2. **新建项目表单**：默认隐藏，点击"+ 新建项目"后在侧边栏内展开。
   - 输入项：名称、目录路径、描述（可选）。
   - 按钮：创建、取消。
   - 创建成功/失败后，表单收起并刷新列表。
3. **项目列表**：
   - 每行显示文件夹图标 + 项目名称。
   - 当前选中行：蓝色背景（`#2563EB`），白色文字。
   - 未选中行：浅灰背景（`#F9FAFB`），深灰文字。
   - 行尾：× 删除按钮，hover 时变红。
   - 空状态：显示"暂无项目，点击上方按钮创建"。
   - 超出侧边栏高度时列表区域独立滚动。

## 交互设计

| 操作 | 触发 | 结果 |
|------|------|------|
| 切换项目 | 点击项目行（除 × 外） | 右侧加载项目配置，该行高亮 |
| 删除项目 | 点击行尾 × | 弹出 `window.confirm` 确认框 |
| 确认删除 | 点击确认框"确定" | Python 删除项目并刷新列表 |
| 取消删除 | 点击确认框"取消" | 无任何操作 |
| 新建项目 | 点击"+ 新建项目" | 侧边栏内展开新建表单 |
| 创建项目 | 点击"创建" | 创建成功，表单收起，列表刷新，新项目置顶 |
| 取消创建 | 点击"取消" | 表单收起 |

## 数据流

```text
┌─────────────────┐      click/select       ┌─────────────────┐
│   gr.HTML 列表   │ ───────────────────────▶ │ 隐藏 gr.Textbox  │
│   (内嵌 JS)     │      click/delete ×      │  (select/delete) │
└─────────────────┘                          └────────┬────────┘
                                                       │ .change()
                                                       ▼
                                              ┌─────────────────┐
                                              │  Python 回调函数  │
                                              │  (ui.py)         │
                                              └────────┬────────┘
                                                       │
                          ┌────────────────────────────┼────────────────────────────┐
                          │                            │                            │
                          ▼                            ▼                            ▼
                   ┌─────────────┐            ┌─────────────┐               ┌─────────────┐
                   │ ProjectStore │            │ ProjectStore │               │ 重新渲染列表  │
                   │   load_project│            │ delete_project│              │ (HTML update)│
                   └─────────────┘            └─────────────┘               └─────────────┘
```

### 关键组件

- `project_list_html`：根据 `ProjectStore.list_projects()` 和当前选中 ID 生成项目列表 HTML。
- `_project_select_bridge`（隐藏 `gr.Textbox`）：接收选中的项目 ID。
- `_project_delete_bridge`（隐藏 `gr.Textbox`）：接收要删除的项目 ID。
- `_on_bridge_select(project_id)`：加载项目并更新右侧表单。
- `_on_bridge_delete(project_id)`：调用 `store.delete_project(project_id)`，刷新列表，清空右侧或自动选中相邻项目。
- `_refresh_project_list(selected_id)`：重新生成 HTML 并更新 `gr.HTML`。

### 安全与边界

- 删除前必须使用原生 `confirm()`，防止误删。
- 删除当前选中的项目后，建议自动选中列表中的下一个项目；如果列表为空，则清空右侧工作区并显示未选择状态。
- 删除失败时通过 `status_msg` 显示错误信息。
- HTML 中对项目名称进行转义，防止 XSS。

## 实现范围

需要修改的文件：

- `app/ui.py`：重构左侧项目管理区，新增 HTML 列表渲染、JS 事件桥、回调函数。
- `app/ui_style.py`：新增项目列表相关 CSS（`.onerad-project-list`、`.onerad-project-item`、`.onerad-project-item-active`、`.onerad-project-delete` 等）。
- `tests/test_ui.py`：补充新交互的测试（列表渲染、选择、删除）。

不修改的文件：

- `app/projects.py`：数据层 API 已足够（`list_projects`、`load_project`、`delete_project`、`create_project`）。
- 右侧工作区配置表单：保持现有组件和事件绑定。

## 测试考虑

- 单元测试 `test_ui.py` 中模拟 `ProjectStore`，验证：
  - 列表渲染包含所有项目名称。
  - 选择项目后右侧表单值正确。
  - 删除项目后 `store.delete_project` 被调用，列表刷新。
- 由于涉及 JS 和 Gradio 事件桥，核心逻辑通过 Python 回调覆盖；浏览器端点击行为可通过手动 smoke 测试验证。

## 风险

- Gradio 的 `gr.HTML` 中嵌入 JS 可能在某些浏览器/Gradio 版本中受 CSP 限制。使用原生 `confirm()` 和简单的 `document.getElementById` 较安全。
- Gradio 升级后 DOM 结构可能变化，但本方案主要依赖隐藏组件的事件桥，HTML 部分只影响样式。
- 项目名称包含特殊字符时需确保 HTML/JS 转义正确。
