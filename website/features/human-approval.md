# 审批面板与人机协同

OneRad 的核心设计原则是**安全可控**：所有有副作用的操作（写文件、跑脚本、耗时计算）都必须经过你确认才会执行，绝不静默执行。

## 审批触发场景

Agent 在不同操作前弹出对应的审批面板：

| 场景 | 面板 | 展示内容 |
|------|------|---------|
| DICOM 转换 / 修改配置（YAML / JSON） | 命令确认面板 | 命令详情与参数 |
| 文件操作（移动 / 复制 / 重命名 / 建目录） | 计划编辑器 | 可编辑的操作计划表格 |
| Python 脚本执行 | 脚本面板 | 代码高亮 + 风险等级标记 |
| 影像 / Mask 配对 | 影像组学面板 | 高 / 中 / 低置信度配对列表 |
| 特征提取 | 影像组学面板 | YAML 路径、输出目录、病例数、预期产物 |
| 建模分析 | 影像组学面板 | 特征 CSV、临床表、ID 列、Label 列 |
| 特征统计检验 | 影像组学面板 | 统计参数（t 检验 / Mann-Whitney U） |
| 子 Agent 分派（写操作模式） | 审批面板 | 子任务列表 |

## 每个面板上的三种操作

| 操作 | 效果 |
|------|------|
| **确认** | 执行该操作，结果返回给 Agent，流程继续 |
| **取消** | 终止该操作。Agent 会收到「用户已取消」的信号，**不会自动重试**，转而询问你想做什么 |
| **其他** | 输入替代指令（如「把输出目录改到 results/ 再执行」），Agent 按你的指令调整 |

::: tip 取消不会导致混乱
取消重操作后，Agent 会明确收到「请不要重试」的提示，避免它把「取消」误读为「执行失败」而反复尝试。
:::

## 计划编辑器：文件操作可改后再批

当 Agent 需要整理文件时，它不会直接动手，而是先生成一份**操作计划**（每条含：动作 / 源路径 / 目标路径 / 理由），你可以在计划编辑器里：

- 删除不需要的条目
- 修改目标路径
- 补充新条目

确认后的执行会自动备份到 `.onerad_backup/`，可回滚。

## 脚本面板：风险分级

Agent 生成的 Python 脚本在审批时带有 **AST 静态分析的风险等级**：

| 等级 | 判定 | 执行方式 |
|------|------|---------|
| 🟢 **low** | 仅用白名单模块（math / json / csv 等），无写操作、无网络、无子进程 | 确认后直接运行 |
| 🟡 **medium** | 引入第三方模块（numpy / pandas 等）、写文件、含路径遍历或绝对路径 | 注入沙箱头（限制文件访问范围）后运行 |
| 🔴 **high** | 涉及网络、子进程、动态执行（eval / exec / os.system）等 | 明确标记风险，经你知情确认后运行 |

## 自动审批开关

对话框附近有**自动审批**开关：

- **关闭（默认）**：每个需确认操作都弹面板，适合初次使用和关键分析
- **开启**：所有审批自动放行，适合已经验证过的批量流程

::: warning
开启自动审批后，文件修改和脚本执行不再有人工拦截。请只在你信任当前任务流程时开启。
:::

## 安全边界

除了审批机制，还有两层底层防护：

- **路径沙箱**：所有文件操作被限制在项目目录内，越界路径（如 `../..`）会被直接拒绝
- **API Key 不落库**：密钥保存在服务内存中，不写入 SQLite 数据库和对话存档
