# 项目记忆

项目记忆是**跨对话的记忆系统**：在一个对话里确认过的事实，同一项目下的所有新对话自动共享，无需重复确认。

## 解决什么问题

没有项目记忆时，每次新建对话都要重新告诉 Agent：

- 「患者 ID 列是 PATIENT_ID」
- 「Label 列 1 是恶性、0 是良性」
- 「我习惯用五折交叉验证」

有了项目记忆，这些**一次确认，项目内永久生效**。

## 工作原理

1. 对话中你确认的关键事实（列名、参数偏好、项目结构认知）被写入项目级存储
2. 新建对话时，Agent 自动读取项目记忆并注入上下文
3. Agent 一开口就知道项目情况，直接接着干活

## 实际效果

**第一个对话**里你确认了 ID 列、Label 列和交叉验证偏好。

**新对话**里你只说：

> 「用同样的数据再做一次分析」

Agent 直接就知道：项目结构、上次确认的列名、你偏好五折交叉验证——不需要任何重复问答。

## 记忆的内容类型

| 类型 | 示例 |
|------|------|
| 数据结构事实 | ID 列名、Label 列定义、协变量列表 |
| 分析偏好 | 交叉验证折数、建模方法偏好 |
| 项目认知 | 目录结构、数据规模、已完成的分析步骤 |

::: tip 记忆与普通历史的区别
普通对话历史只在单个对话线程内有效；项目记忆是**项目级**的，跨线程共享。删除某个对话不会影响项目记忆。
:::
