# 断点续提

特征提取是影像组学流程中最耗时的环节（30 例约十几分钟，上百例可能要几小时）。断点续提机制保证：**任何中断都不会浪费已完成的计算**。

## 工作机制

- 每例病例的特征提取完成后，立即写入 **h5 缓存**
- 下次启动提取时，Agent 逐例检查：已有 h5 缓存**且提取参数未变** → 直接跳过
- 只有新病例或参数变更的病例才会真正执行提取

## 覆盖的中断场景

| 场景 | 恢复方式 |
|------|---------|
| 手动点了「停止」 | 说「继续分析」，从断点接着跑 |
| 不小心关了浏览器页面 | 重新打开，说「继续分析」 |
| 服务 / 容器重启 | 重启后说「继续分析」 |
| 新增了病例 | 只对新增病例提取，老病例自动跳过 |

## 使用示例

提取到一半中断后，新对话或直接输入：

> **继续分析**

Agent 发现 `radiomics_features.csv` 或 h5 缓存已存在，自动跳过已完成的病例，从断点继续。

## 强制重新提取

如果修改了 YAML 参数（如 binWidth），参数变化会被自动检测，相关缓存失效。

如果想无视缓存全部重提，直接说：

> **强制重新提取所有病例**

::: warning
强制重提会丢弃全部已有缓存，大样本项目请谨慎使用。
:::
