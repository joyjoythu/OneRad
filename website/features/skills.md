# 分析技能（Skills）

OneRad 的专业能力不是靠一个巨大的提示词，而是靠一组**按需加载的技能模块**（Skills）。每个 Skill 是一份领域操作规范，Agent 在执行对应任务时加载它，就像研究者查阅标准操作流程（SOP）。

## 内置技能一览

| 技能 | 作用 | 何时生效 |
|------|------|---------|
| **agent-core** | Agent 的核心行为规范：安全原则、工具使用约定、确认语义 | 每轮对话 |
| **radiomics-workflow** | 影像组学全流程指引：配对发现 → 参数确认 → 提取 → 临床匹配 → 建模 → 报告 | 每轮对话 |
| **clinical-columns** | 从临床表格中识别患者 ID 列、二分类结局列和可用临床特征列 | 解析临床表时 |
| **filename-id** | 从影像 / 掩膜文件名样例中推断患者 ID 的正则表达式 | 配对发现时 |
| **file-operations** | 生成安全的文件整理计划（JSON 格式，仅允许移动 / 复制 / 重命名 / 建目录） | 整理项目文件时 |
| **report-writing** | 润色报告的方法学段落，同时严格保留所有事实数值 | 撰写报告时 |
| **result-interpretation** | 基于数值摘要生成中文结果解读（模型性能 / 特征意义 / SHAP） | 分析完成后 |
| **word-report** | Word 文档格式规范及文档工具使用指引 | 生成 / 重排报告时 |
| **thread-title** | 为新对话生成简短的中文标题 | 对话首轮 |

## 技能机制的特点

**热更新友好**：技能文件（`SKILL.md`）每次调用时从磁盘读取，无缓存。修改技能内容后**立即生效**，无需重启服务——你可以按自己的研究习惯调整 Agent 的行为规范。

**分层加载**：核心技能（agent-core、radiomics-workflow）每轮对话都生效，保证行为一致性；任务技能只在对应场景加载，不占用无关轮次的上下文。

**约束而非脚本**：技能告诉 Agent「应该怎么做」（如配对算法的置信度分级规则、临床列的识别标准），具体执行仍由 Agent 根据项目实际情况决策。
