# 并行子 Agent 探索

一次完整的影像组学分析需要了解项目里的大量信息：目录结构、文件清单、YAML 配置、临床表格列名、图像 / Mask 配对情况。如果这些信息串行地在主对话里一步步收集，中间结果会迅速占满上下文窗口。

OneRad 的解决方案：**子 Agent 系统**。

## 核心思想

1. **并行加速** — 多个独立探索任务同时运行（最多 4 个并发）
2. **上下文隔离** — 子 Agent 的中间过程不进主对话，只有结论返回
3. **安全分级** — 只读探索免确认，写操作模式需确认

## 两种模式

| 维度 | 只读探索（explore） | 全功能（general） |
|------|--------------------|--------------------|
| 工具集 | 仅 8 个只读工具 | 几乎全部工具 |
| 审批流程 | 免确认，直接执行 | 需确认 |
| 典型用途 | 开局探索：扫目录 / 读配置 / 查临床表 | 需要写操作或计算的子任务 |
| 可见性 | 中间过程只推前端卡片 | 中间过程只推前端卡片 |

## 界面上看到什么

子 Agent 运行时，界面出现**状态卡片**，滚动展示每个子 Agent 最近的行动摘要（读哪个文件、查什么配置）。完成后卡片展示最终结论。

## 隔离的价值

假设探索一个项目需要读 10 个文件、产生 2 万 token 的中间内容：

- **没有隔离**：2 万 token 全部进入主对话历史，后续每一步 LLM 调用都要带着它们，又贵又慢
- **有隔离**：只有「项目有 30 对图像 / Mask、YAML 的 binWidth=25、临床表 ID 列是 PATIENT_ID」这样的结论（几百 token）回到主对话

子 Agent 的结论会被截断在 4000 字符以内，保证主对话始终精简。

## 取消传播

点击「停止」时，取消信号会传播到所有正在运行的子 Agent：

- 子 Agent 每完成一轮工具调用检查一次取消信号
- 特征提取等耗时操作每完成一个病例检查一次
- 检测到取消立即退出，已完成的进度保留
