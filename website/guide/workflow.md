# 完整分析流程

本页详细拆解 OneRad 影像组学全流程的七个环节，以及每个环节你需要做什么。

```
项目探索 → 配对发现 → 参数确认 → 特征提取 → 临床表审查 → 建模分析 → 解读与报告
```

## 环节一：项目探索

Agent 的第一步永远是**探索**：并行派出多个只读子 Agent，扫描目录结构、检查图像和 Mask 配对情况、读取提取参数、识别临床表格。

- 中间过程只推送到前端的子 Agent 卡片，**不污染主对话上下文**，只有结论回到主对话
- 探索是**增量分析的基础**——Agent 先看清楚项目里有什么，再决定从哪一步开始
- 详见 [并行子 Agent 探索](/features/subagents)

**你要做的**：什么都不用做，等待探索结论。

## 环节二：配对发现

Agent 自动把 `images/` 和 `masks/` 按文件名匹配（[配对算法](/guide/data-preparation#影像与掩膜的命名)），给出高 / 中 / 低置信度的配对列表，以及未匹配的影像和掩膜清单。

**你要做的**：在配对确认面板检查配对结果（重点看中低置信度项），确认后进入下一步；发现问题点「取消」并告诉 Agent 哪里不对。

## 环节三：参数确认

Agent 自动检查图像的 spacing，与 YAML 配置（`resampledPixelSpacing` 等）比对。如果实测 spacing 和配置不一致，它会**建议调整参数**，而不是盲目按错误参数跑。

**你要做的**：确认提取参数（YAML 路径、输出目录、病例数、预期产物）。想改参数可以直接说，如「把 binWidth 改成 25」。

## 环节四：特征提取

基于 PyRadiomics 的高通量特征提取：

- **断点续提**：每例提取完写入 h5 缓存，下次遇到相同参数直接跳过。中途停止、页面关闭、服务重启都不怕
- **逐例进度条**：界面实时显示 `当前病例 / 总数`
- 30 例数据约需十几分钟（取决于特征类别和机器性能）

**你要做的**：确认后等待。想中断点「停止」，下次说「继续分析」即可从断点接着跑。详见 [断点续提](/features/resume)。

## 环节五：临床表审查

Agent 自动识别临床表结构：

- 患者 ID 列（用于把特征表和临床表对齐）
- Label 列（0/1 结局变量）
- 协变量（可选）
- **中文列名自动翻译成英文**，避免后续 SHAP 图乱码

识别结果有歧义时，Agent 会主动向你提问（结构化选择面板），而不是瞎猜。

**你要做的**：确认或纠正列的识别结果。确认过的事实记入[项目记忆](/features/project-memory)。

## 环节六：建模分析

确认分析参数后执行建模流水线：

| 步骤 | 方法 |
|------|------|
| 特征筛选 | LASSO（L1 正则化） |
| 建模 | 逻辑回归 |
| 验证 | 分层五折交叉验证（StratifiedKFold） |
| 评估 | 每折输出 ROC、校准曲线、DCA |
| 可解释性 | SHAP 特征贡献分析 |

**你要做的**：确认分析参数（特征 CSV、临床表、ID 列、Label 列），等待完成。

## 环节七：解读与报告

分析完成后 Agent 自动执行两件事：

1. **三段式中文解读**——`【模型性能解读】`（AUC / 敏感度 / 特异度 / 逐折稳定性）、`【特征意义解读】`（系数 / OR / SHAP 方向一致性）、`【SHAP 可解释性解读】`（top 特征贡献方向与幅度）
2. **生成 Word 报告**——中文学术格式，含方法学描述、结果表格和全部图表，可直接放进论文 Supplementary

**你要做的**：阅读解读和报告。格式不满意可以说「**重排报告**」，详见 [Word 报告生成](/features/word-report)。

## 增量分析与重复实验

OneRad 是**增量式**的——Agent 会检测已有产物并跳过已完成步骤：

| 你说 | Agent 做 |
|------|---------|
| 「继续分析」 | 发现 `radiomics_features.csv` 已存在 → 跳过提取，从建模继续 |
| 「新增了 5 个病例，继续」 | 只对新增病例提取特征（断点续提），再重新建模 |
| 「用同样的数据再做一次分析」 | 从项目记忆读取已知列名和偏好，快速重跑 |
| 「强制重新提取」 | 忽略 h5 缓存，全部重提 |

## 全程通用操作

| 操作 | 位置 | 效果 |
|------|------|------|
| 停止 | 停止按钮 | 立即中断当前执行（含运行中的子 Agent），已完成的进度保留 |
| 取消审批 | 审批面板「取消」 | 终止该操作，Agent 不会自动重试，会询问你接下来想做什么 |
| 自动审批 | 自动审批开关 | 开启后需确认的操作自动放行（适合可信的批量流程） |
| @ 文件索引 | 输入框输入 `@` | 建立可搜索的项目文件索引，快速定位产物文件 |
