# 常见问题 FAQ

## 部署与启动

### 8000 端口被占用怎么办？

修改 `.env` 中的 `PORT`（如 `PORT=8080`），然后 `docker compose up -d` 重启。

### 首次构建镜像很慢？

首次构建约 5–10 分钟（下载基础镜像 + 安装依赖 + 编译前端），属正常现象。后续构建有缓存只需几十秒。如果卡在某个包下载，检查网络或配置镜像加速。

### API Key 配在哪？

两种方式任选：

1. `.env` 文件中的 `DEEPSEEK_API_KEY`（启动前配置）
2. Web 界面**设置页**中填入（启动后配置，优先级更高）

API Key 只保存在服务内存中，不会写入数据库。

### 容器删了数据会丢吗？

不会。数据库（`./data`）和分析产物（`./output`）都挂载在宿主机目录，删除容器不影响数据。

## 使用过程

### 提取中途停止了怎么办？

什么都不用担心。每例提取完成即写入 h5 缓存，下次说「继续分析」自动从断点接着跑。详见 [断点续提](/features/resume)。

### 我点了「取消」，Agent 会不会自己重试？

不会。取消重操作后，Agent 会收到「用户已取消，请不要重试」的明确信号，转而询问你想做什么。

### Agent 识别的 ID 列 / Label 列不对？

在确认面板直接点「取消」，然后告诉它正确的列名，如「ID 列是 patient_no」。确认过的列名会记入[项目记忆](/features/project-memory)，以后不再问。

### 怎么强制重新提取全部特征？

直接说「强制重新提取所有病例」。注意这会丢弃全部 h5 缓存。详见 [断点续提](/features/resume#强制重新提取)。

### 报告格式乱了怎么办？

说「重排报告」，Agent 会把最新的分析报告重新套版为中文学术格式。详见 [Word 报告生成](/features/word-report)。

## 安全与控制

### 自动审批开关是什么？安全吗？

开启后所有需确认的操作（文件修改、脚本执行、特征提取等）自动放行。适合已验证的批量流程。初次使用建议保持关闭。详见 [审批面板](/features/human-approval#自动审批开关)。

### Agent 会动我项目目录以外的文件吗？

不会。所有文件操作被路径沙箱限制在项目目录内，越界路径直接被拒绝。
