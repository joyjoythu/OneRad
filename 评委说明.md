# OneRad — 评委使用说明

## 前置条件

电脑需安装 Docker Desktop：https://www.docker.com/products/docker-desktop/

---

## 第一步：导入镜像（只需一次）

打开终端（CMD / PowerShell / Terminal），进入本说明文件所在目录，执行：

```bash
docker load < autoradiomics.tar.gz
```

---

## 第二步：设置 API Key

**Windows CMD：**
```cmd
set DEEPSEEK_API_KEY=sk-你的key
```

**Windows PowerShell：**
```powershell
$env:DEEPSEEK_API_KEY="sk-你的key"
```

**Mac / Linux：**
```bash
export DEEPSEEK_API_KEY=sk-你的key
```

---

## 第三步：配置影像数据路径

用记事本（或任意文本编辑器）打开 `docker-compose.offline.yml`，找到这一行：

```yaml
- D:\你的影像数据目录:/data/input
```

把冒号左边的路径改成你电脑上影像数据所在的**实际目录**，例如：

```yaml
# Windows 示例
- D:\MedicalImages:/data/input

# Mac / Linux 示例
- /home/judge/images:/data/input
```

> 冒号**左边**是你的电脑路径，**右边** `/data/input` 保持不变。

---

## 第四步：启动

```bash
docker compose -f docker-compose.offline.yml up -d
```

---

## 第五步：使用

1. 浏览器打开 http://localhost:8000
2. 左侧点击 **+ 新建项目**
3. 点击 **浏览**，左侧位置列表会出现 **📂 /data/input**
4. 进入后选择你的影像数据目录，确认
5. 在对话中告诉 AI 你要做什么分析

---

## 停止

```bash
docker compose -f docker-compose.offline.yml down
```

---

## 常见问题

**Q: 浏览器打开后页面空白或报错？**
A: 等待 10 秒后刷新页面，容器启动需要时间。

**Q: 创建项目时提示"路径已被使用"？**
A: 该路径已创建过项目，在左侧边栏右键删除旧项目后重新创建即可。

**Q: 文件选择器看不到我的数据？**
A: 检查 `docker-compose.offline.yml` 中的宿主机路径是否正确，修改后执行 `docker compose -f docker-compose.offline.yml down && docker compose -f docker-compose.offline.yml up -d` 重启。
